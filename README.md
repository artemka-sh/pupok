## 1. Цель работы

Программа моделирует **согласующую инжекционную линию** — участок ускорительного комплекса, по которому пучок заряженных частиц поступает из линейного инжектора в кольцо коллайдера. Основная задача — подобрать такие параметры квадрупольных магнитов, чтобы оптические (Twiss) параметры пучка на выходе из линии точно совпали с параметрами, требуемыми кольцом (условие «матчинга»). Одновременно моделируется геометрическая траектория центроида пучка с учётом дипольных магнитов (септума и кикера).

## 2. Физическая модель и допущения

### 2.1 Матричная оптика (трансфер-матрицы)

Движение частиц описывается **формализмом матриц переноса** в приближении линейной оптики. Каждый элемент линии задаётся матрицей 2×2, действующей на вектор состояния частицы `(x, x')` (поперечное смещение и угол):

| Элемент | Матрица переноса |
|---|---|
| Дрейфовый промежуток длиной `L` | `[[1, L], [0, 1]]` |
| Фокусирующий квадруполь (x-плоскость) | `[[cos(√k·L), sin(√k·L)/√k], [-√k·sin(√k·L), cos(√k·L)]]` |
| Дефокусирующий квадруполь | матрица с `cosh/sinh` |

Плоскости X и Y рассматриваются **независимо**: квадруполь, фокусирующий в X, одновременно дефокусирует в Y (знак `K` меняется).

### 2.2 Twiss-параметры (бета-функции)

Оптика пучка в каждой точке линии описывается **Twiss-параметрами** `β` (бета-функция) и `α` (альфа-функция), задающими форму и ориентацию фазового эллипса. В коде они хранятся в виде симметричной матрицы:

```
T = | β   -α |
    | -α   γ |    где γ = (1+α²)/β
```

Эволюция матрицы T вдоль линии:

```cpp
Matx22d transformTwiss(const Matx22d& T_in, const Matx22d& M) {
    return M * T_in * M.t();
}
```

Это соответствует формуле **σ' = M σ Mᵀ** для матрицы пучка.

### 2.3 Допущения модели

- **Линейное приближение** — нелинейные поля магнитов (секступоли, октуполи) не учитываются.
- **Нет связи плоскостей** — движение в X и Y полностью развязано (нет наклонных квадруполей, нет solenoid-поля).
- **Нет потерь частиц** — все трекируемые частицы проходят линию без столкновений со стенками.
- **Нет хроматики** — все частицы считаются с одним импульсом (нет энергетического разброса).
- **Дипольный кикер идеален** — моделируется мгновенным обнулением угла и смещения центроида в точке инжекции.
- **Нормализованный эмиттанс** зафиксирован: `εₓ = 10 нм·рад`, `εᵧ = 2 нм·рад`.

## 3. Конфигурация магнитной системы

### 3.1 Почему именно такая схема

Общая длина согласующей линии составляет **7.0 м**, разбитых на 5 дрейфовых промежутков и 4 квадруполя:

```
[ DRIFT 1m ] [ Q1 0.5m ] [ DRIFT 1m ] [ Q2 0.5m ] [ DRIFT 1m ] [ Q3 0.5m ] [ DRIFT 1m ] [ Q4 0.5m ] [ DRIFT 1m ]
```

Четыре квадруполя — минимально необходимое количество, чтобы одновременно совместить **4 независимых параметра** на выходе: `βₓ`, `αₓ`, `βᵧ`, `αᵧ`. Каждый квадруполь даёт одну степень свободы (градиент поля `K`), поэтому система квадратная (4 уравнения — 4 неизвестных).

### 3.2 Целевые параметры на входе в коллайдер

| Параметр | Целевое значение |
|---|---|
| `βₓ` | 8.0 м |
| `αₓ` | 0.0 (пучок в «талии») |
| `βᵧ` | 4.0 м |
| `αᵧ` | 0.0 (пучок в «талии») |

Условие `α = 0` означает, что в точке инжекции пучок находится в перетяжке (waist) — это стандартное требование коллайдера для обеспечения минимального расходимости.

### 3.3 Начальные параметры пучка (вход в линию)

```cpp
Matx22d Tx(5.0,  0.5,  0.5,  1.25/5.0);   // βx=5.0 м, αx=-0.5
Matx22d Ty(2.5, -0.3, -0.3,  1.09/2.5);   // βy=2.5 м, αy=+0.3
```

### 3.4 Дипольные магниты

**Септум-магнит (s = 0 м):**
Пучок начинает движение со смещением `centroid_x = 0.1 м` (10 см от оси коллайдера) и под углом `initial_angle = -0.014 рад` (−14 мрад). Это имитирует отклонение пучка от оси инжектора диполем-септумом в сторону кольца.

**Кикер (s ≈ 6.9 м):**
В точке инжекции (конец линии) выполняется мгновенная коррекция:
```cpp
if (s_curr >= total_L - 0.1) {
    centroid_x = 0;
    initial_angle = 0;
}
```
Кикер «обнуляет» оба параметра центроида — пучок встаёт на целевую орбиту коллайдера. Без него пучок пересёк бы кольцо по диагонали и потерялся.

## 4. Архитектура кода

### 4.1 Основные структуры данных

```cpp
struct Particle {
    double x, xp;   // поперечное смещение [м] и угол [рад] в X
    double y, yp;   // то же в Y
};

struct BeamState {
    double s;                    // позиция вдоль линии [м]
    double bx, ax, by, ay;       // Twiss-параметры (β и α) в X и Y
    double centroid_x;           // смещение центра пучка [м]
    bool is_magnet;              // флаг: находимся внутри квадруполя
    vector<Particle> particles;  // снимок ансамбля частиц в данной точке
};
```

`history` — глобальный вектор `BeamState`, в который записывается состояние системы через каждый шаг `ds = 0.005 м`. Именно он служит источником данных для всей интерактивной визуализации.

### 4.2 Оптимизатор

Для подбора градиентов квадруполей `K[0..3]` используется **метод Нелдера–Мида** (Downhill Simplex) из библиотеки OpenCV:

```cpp
Ptr<DownhillSolver> solver = DownhillSolver::create();
solver->setFunction(makePtr<BeamMatchingObjective>());
```

Целевая функция — сумма квадратов отклонений выходных Twiss-параметров от целевых:

```cpp
return pow((Tx_out(0,0) - 8.0), 2) + pow(-Tx_out(0,1) - 0.0, 2) +
       pow((Ty_out(0,0) - 4.0), 2) + pow(-Ty_out(0,1) - 0.0, 2);
```

Начальная точка: `K = [0.5, -0.5, 0.5, -0.5]` м⁻². Критерий сходимости: 50 000 итераций или ошибка < 10⁻¹⁰.

## 5. Ключевые участки кода

### 5.1 Расчёт матрицы квадруполя

```cpp
Matx22d getQuad(double L, double K, bool is_x) {
    if (std::abs(K) < 1e-9) return getDrift(L);  // нет поля — дрейф
    double k_val = is_x ? K : -K;                // знак меняется для Y
    double r = std::sqrt(std::abs(k_val));
    if (k_val > 0) {
        // фокусирующий — тригонометрические функции
        return Matx22d(cos(r*L), (1.0/r)*sin(r*L),
                      -r*sin(r*L), cos(r*L));
    } else {
        // дефокусирующий — гиперболические функции
        return Matx22d(cosh(r*L), (1.0/r)*sinh(r*L),
                       r*sinh(r*L), cosh(r*L));
    }
}
```

Функция единая для обеих плоскостей: параметр `is_x` управляет знаком `K`.

### 5.2 Генерация ансамбля частиц

```cpp
vector<Particle> generateBeam(int N, double betaX, double alphaX,
                               double betaY, double alphaY) {
    RNG rng(12345);
    for (int i = 0; i < N; i++) {
        double r = sqrt(rng.uniform(0.0, 1.0));    // равномерное по площади
        double theta = 2 * CV_PI * rng.uniform(0.0, 1.0);

        double x  =  sqrt(eps_x * betaX) * r * cos(theta);
        double xp = -sqrt(eps_x / betaX) * (alphaX * r*cos(theta) + r*sin(theta));
        // аналогично для Y ...
    }
}
```

1000 частиц генерируются **равномерно внутри фазового эллипса** (координата `r` берётся как `√(uniform)`, а не `uniform` — иначе частицы скапливались бы у центра). Форма эллипса задаётся начальными Twiss-параметрами.

### 5.3 Трекинг (основной цикл)

```cpp
auto track = [&](double L, double k) {
    for (double t = 0; t < L; t += ds) {

        // 1. Движение центроида под действием диполей
        centroid_x += initial_angle * ds;

        // 2. Кикер в конце линии
        if (s_curr >= total_L - 0.1) {
            centroid_x = 0;
            initial_angle = 0;
        }

        // 3. Сохранение снимка состояния
        history.push_back({s_curr, Tx(0,0), -Tx(0,1), Ty(0,0), -Ty(0,1),
                            centroid_x, std::abs(k) > 1e-6, beam});

        // 4. Шаг матрицы переноса для Twiss и для частиц
        Matx22d Mx_step = getQuad(ds, k, true);
        Matx22d My_step = getQuad(ds, k, false);
        Tx = transformTwiss(Tx, Mx_step);
        Ty = transformTwiss(Ty, My_step);
        propagateParticles(beam, Mx_step, My_step);

        s_curr += ds;
    }
};
```

Трекинг вызывается последовательно для каждого элемента линии:

```cpp
track(L_drift, 0);                                    // дрейф перед Q1
for (int i = 0; i < 4; ++i) {
    track(L_quad, K[i]);   // квадруполь Qi
    track(L_drift, 0);     // дрейф после Qi
}
```

### 5.4 Целевая функция оптимизатора

```cpp
class BeamMatchingObjective : public MinProblemSolver::Function {
public:
    int getDims() const override { return 4; }
    double calc(const double* x) const override {
        // строим полную матрицу переноса всей линии
        for (int i = 0; i < 4; ++i) {
            Mx = getQuad(L_quad, x[i], true) * getDrift(L_drift) * Mx;
            My = getQuad(L_quad, x[i], false) * getDrift(L_drift) * My;
        }
        // применяем её к начальным Twiss-параметрам
        Matx22d Tx_out = transformTwiss(Tx, Mx);
        Matx22d Ty_out = transformTwiss(Ty, My);

        // возвращаем сумму квадратов невязок с целевыми значениями
        return pow((Tx_out(0,0) - 8.0), 2) + pow(-Tx_out(0,1) - 0.0, 2) +
               pow((Ty_out(0,0) - 4.0), 2) + pow(-Ty_out(0,1) - 0.0, 2);
    }
};
```

Обратите внимание: оптимизатор работает **аналитически** (без пошагового трекинга), умножая матрицы напрямую. Пошаговый трекинг запускается уже с найденными K только для сохранения `history`.

## 6. Визуализация

Программа открывает **5 окон OpenCV**, все связанные общим слайдером по позиции `s`:

### Окно 1 — Beam Envelope (огибающая)

Статический график бета-функций вдоль всей линии. Розовая линия — `βₓ(s)`, жёлтая — `βᵧ(s)`. Вертикальные зелёные полосы — расположение квадруполей.

### Окно 2 — Current Phase Space

Фазовый эллипс `(x, x')` в текущей точке `s`. Розовый эллипс — X-плоскость, жёлтый — Y. Обновляется при движении слайдера.

### Окно 3 — 3D Beam Cross-Section

Поперечный срез пучка в точке `s`: проекция частиц на плоскость `(x, y)`. Фон зелёный = квадруполь, серый = дрейф.

### Окно 4 — Collider Acceptance Matching

Сравнение текущего фазового эллипса пучка (розовый/жёлтый) с целевым эллипсом коллайдера (зелёный). По этому окну видно, насколько успешно прошёл матчинг.

### Окно 5 — Trajectory View (Top-Down)

Вид сверху на траекторию центроида пучка. Красная линия — пройденный путь. Серая полоса — труба коллайдера (целевая орбита). Зелёная вертикальная черта в конце — позиция кикер-магнита.

## 7. Результаты работы

После запуска оптимизатор находит значения `K[0..3]`, при которых невязка целевой функции стремится к нулю. Визуально это означает, что в окне «Collider Acceptance Matching» при `s = 7.0 м` розовый и жёлтый эллипсы совпадают с зелёными целевыми контурами.

Траектория центроида в окне «Trajectory View»:
- стартует с отклонением 10 см от оси,
- плавно сближается с нулём под действием угла септума,
- резко выравнивается в позиции `s ≈ 6.9 м` (кикер).

**Итог:** пучок попадает в кольцо коллайдера с нулевым смещением центроида и согласованными Twiss-параметрами — условия успешной инжекции выполнены.
**Итог:** пучок попадает в кольцо коллайдера с нулевым смещением центроида и согласованными Twiss-параметрами — условия успешной инжекции выполнены.
