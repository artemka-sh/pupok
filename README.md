# 🔬 Beam Matching Simulation

Симуляция согласования пучка заряженных частиц с каналом ускорителя на основе оптики Твисса, матричного трекинга и численной оптимизации.

---

## 📌 Задача

> Согласовать параметры пучка с каналом ускорителя, показать из каких элементов магнитной системы состоит согласующий участок, а также визуализировать изменение эмиттанса пучка вдоль трассы.

Под **согласованием** понимается подбор таких градиентов квадрупольных линз, при которых β-функции пучка на выходе из канала принимают заданные (целевые) значения, а дисперсионная функция Dₓ стремится к нулю.

---

## ⚙️ Состав согласующего участка

Магнитная структура канала состоит из следующих элементов:

| # | Элемент | Тип | Длина | Параметр |
|---|---|---|---|---|
| 1–4 | Дрейфовые промежутки | Drift Space | 1.0 м | — |
| 5–8 | Квадрупольные линзы Q1–Q4 | Quadrupole Magnet | 0.5 м каждая | K (оптимизируется) |
| 9 | Отклоняющий диполь | Injection Dipole | 1.0 м | угол 0.1 рад |

Полная топология трассы:
```
[Drift 1m] → [Q1] → [Drift 1m] → [Q2] → [Drift 1m] → [Q3] → [Drift 1m] → [Q4] → [Drift 1m] → [Dipole]
```

---

## 🧮 Физическая модель и метод решения

### 1. Матрицы переноса

Движение частицы в линейном приближении описывается матрицей переноса 3×3 (дополнительная координата — импульсный разброс `dp`):

```
| x_out  |     | M00  M01  M02 |   | x_in  |
| x'_out |  =  | M10  M11  M12 | × | x'_in |
| dp     |     |  0    0    1  |   | dp    |
```

#### Дрейфовый промежуток (`getDrift`)
Частица летит прямолинейно, угол не меняется:
```cpp
Matx33d getDrift(double L) {
    return Matx33d(1.0, L, 0.0,   // x_out = x + L·x'
                   0.0, 1.0, 0.0, // x'_out = x' (угол не меняется)
                   0.0, 0.0, 1.0);
}
```

#### Квадрупольная линза (`getQuad`)
Фокусирует в одной плоскости и дефокусирует в другой.  
Знак `k_val` меняется для вертикальной плоскости (`is_x = false`):

```cpp
// Фокусирующая плоскость (k > 0) — синусоидальное решение:
Matx33d(cos(r*L),        (1/r)*sin(r*L),  0,
        -r*sin(r*L),      cos(r*L),        0,
        0,                0,               1)

// Дефокусирующая плоскость (k < 0) — гиперболическое решение:
Matx33d(cosh(r*L),       (1/r)*sinh(r*L), 0,
        r*sinh(r*L),      cosh(r*L),       0,
        0,                0,               1)
```

#### Отклоняющий диполь (`getDipole`)
Описывает движение по дуге радиуса `ρ = L / angle`.  
Вертикальная плоскость — простой дрейф (без магнитного отклонения):

```cpp
Matx33d(cos(angle),          rho*sin(angle),   rho*(1 - cos(angle)),
        -(1/rho)*sin(angle),  cos(angle),       sin(angle),
        0,                    0,                1)
```
Третий столбец (`M02`, `M12`) — вклад дисперсионной функции: при `dp ≠ 0` частица отклоняется на иной угол, что порождает Dₓ.

### 2. Параметры Твисса и транспортировка огибающей

Параметры пучка (β, α) хранятся в матрице Куранта-Снайдера 2×2:

```
T = | β      -α        |
    | -α   (1+α²)/β    |
```

Транспортировка через элемент с матрицей M:
```cpp
Matx22d transformTwiss(const Matx22d& T_in, const Matx33d& M3) {
    Matx22d M(M3(0,0), M3(0,1), M3(1,0), M3(1,1)); // Берём 2x2 подматрицу
    return M * T_in * M.t();                          // T_out = M · T_in · Mᵀ
}
```

Начальные условия (на входе в канал):
- Горизонталь: `βₓ = 5.0 м`, `αₓ = -0.5`
- Вертикаль: `βᵧ = 2.5 м`, `αᵧ = 0.3`

Целевые значения (на выходе из канала):
- `βₓ → 8.0 м`, `αₓ → 0`
- `βᵧ → 4.0 м`, `αᵧ → 0`
- `Dₓ → 0` (дисперсия подавлена)

---

### 3. Оптимизация — метод Нелдера-Мида

Для подбора градиентов K1–K4 используется **симплекс-метод (Downhill Solver)** из OpenCV.

#### Целевая функция (`BeamMatchingObjective::calc`):

```cpp
double calc(const double* x) const override {
    // x[0..3] — градиенты K1, K2, K3, K4

    // 1. Строим полную транспортную матрицу канала
    for (int i = 0; i < 4; ++i) {
        Mx = getQuad(L_quad, x[i], true)  * getDrift(L_drift) * Mx;
        My = getQuad(L_quad, x[i], false) * getDrift(L_drift) * My;
    }
    Mx = getDrift(L_drift) * Mx;
    Mx = getDipole(L_dipole, dip_angle, true) * Mx;

    // 2. Транспортируем параметры Твисса через весь канал
    Matx22d Tx_out = transformTwiss(Tx, Mx);
    Matx22d Ty_out = transformTwiss(Ty, My);

    // 3. Минимизируем отклонение от целевых β-функций + штраф за дисперсию
    double w = 1000.0; // Большой штраф — дисперсия должна быть ≈ 0
    return pow(Tx_out(0,0) - 8.0, 2)    // βx → 8 м
         + pow(Tx_out(0,1) - 0.0, 2)    // αx → 0
         + pow(Ty_out(0,0) - 4.0, 2)    // βy → 4 м
         + pow(Ty_out(0,1) - 0.0, 2)    // αy → 0
         + w * (D(0)*D(0) + D(1)*D(1)); // Dₓ → 0
}
```

Настройки решателя:
```cpp
solver->setTermCriteria(TermCriteria(
    TermCriteria::MAX_ITER + TermCriteria::EPS,
    50000,   // Максимум итераций
    1e-16    // Точность сходимости
));
```

### 4. Генерация пучка (`generateBeam`)

800 частиц генерируются с распределением, соответствующим фазовому эллипсу заданного эмиттанса.  
Используется метод **инверсии площади** (равномерное покрытие эллипса):

```cpp
double r = sqrt(u);              // √u для равномерного заполнения площади
double theta = 2 * M_PI * v;

// Координаты на фазовом эллипсе с учётом параметров Твисса:
double x  =  sqrt(eps_x * betaX) * r * cos(theta);
double xp = -sqrt(eps_x / betaX) * (alphaX * r * cos(theta) + r * sin(theta));
```

Импульсный разброс `dp` генерируется как гауссово распределение с σ = 10⁻³:
```cpp
double dp = rng.gaussian(1e-3);
```

Эмиттансы:
- `εₓ = 10 нм·рад`
- `εᵧ = 2 нм·рад`

---

### 5. Покадровый трекинг (`track_drift`, `track_quad`, `track_dipole`)

Трасса разбита на шаги `ds = 5 мм`. На каждом шаге:
1. Текущее состояние пучка сохраняется в `history`
2. Параметры Твисса обновляются через `transformTwiss`
3. Дисперсионный вектор обновляется: `D_vec = M_step * D_vec`
4. Все частицы транспортируются через `propagateParticles`

```cpp
// Трекинг одной частицы через матрицу M:
double x_new  = Mx(0,0)*p.x + Mx(0,1)*p.xp + Mx(0,2)*p.dp;
double xp_new = Mx(1,0)*p.x + Mx(1,1)*p.xp + Mx(1,2)*p.dp;
```

Член `Mx(0,2)*p.dp` — хроматический сдвиг: частицы с разным импульсом разлетаются по-разному (именно это формирует дисперсию).

## 🖥️ Визуализация

Программа открывает три интерактивных окна:

### 📈 Beam Envelope — огибающая пучка
Отображает β-функции и дисперсию вдоль всей трассы.

| Цвет | Значение |
|---|---|
| 🔴 Красная кривая | βₓ — горизонтальная огибающая |
| 🟡 Жёлтая кривая | βᵧ — вертикальная огибающая |
| 🟢 Зелёная линия | Дисперсия Dₓ |
| Зелёный фон | Квадруполь |
| Синеватый фон | Диполь |

### 🔵 Phase Space — фазовое пространство
Показывает фазовые эллипсы пучка в текущей позиции S (обновляется ползунком).

| Цвет | Значение |
|---|---|
| 🔴 Красный эллипс | Горизонтальная плоскость (x, x') |
| 🔵 Синий эллипс | Вертикальная плоскость (y, y') |
| 🟢 Зелёная точка | Центр — дисперсионная функция |

Размер эллипса пропорционален `√(ε·β)` — чем больше β-функция, тем шире пучок. Наклон эллипса определяется α: при `α = 0` эллипс вертикален (оптический перехват).

### ⚪ 3D Beam Cross-Section — сечение пучка
Показывает реальное положение частиц в поперечной плоскости (x, y).

| Цвет фона | Тип элемента |
|---|---|
| Зелёный | Квадруполь |
| Оранжевый | Диполь |
| Серый | Дрейф |

Ползунок **Position S** позволяет пролистывать всю историю состояния пучка от начала до конца трассы.

## 🛠️ Зависимости и сборка

- **C++17**
- **OpenCV 4.x** (матричная алгебра `Matx`, оптимизатор `DownhillSolver`, визуализация `imshow`)

### Сборка

```bash
g++ -std=c++17 main.cpp -o beam_sim \
    $(pkg-config --cflags --libs opencv4)
```

### Запуск

```bash
./beam_sim
```

После запуска программа:
1. Автоматически запускает оптимизацию (~несколько секунд)
2. Открывает три окна визуализации
3. Выводит результаты в консоль
4. Ожидает нажатия любой клавиши для завершения
